# TG Stremio v3.2.0 — Sticky Multi-User Router

This release makes the stream router stable for real playback:

- Each viewer is assigned one healthy Telegram bot for playback, resume, and seeking.
- Extra `MULTI_TOKEN*` / WebUI stream bots are used to serve **other viewers**, not to stripe a single stream.
- Direct and raw split streams use seek-aware `206` windows: a 32 MB cold-play window and 16 MB seek window by default.
- A small same-bot GetFile read-ahead queue preserves low RAM usage and fast first bytes.
- Repeat failures can safely clear stickiness so a later range uses a healthy replacement bot.

## Best upload format

1. Single `Movie.mkv` — fastest.
2. Raw `Movie.mkv.001`, `Movie.mkv.002` — best for files too large for a single Telegram upload.
3. Stored/no-compression ZIP parts — compatible but less ideal.
4. Compressed ZIP parts — playable, but fast seeking is not possible by archive design.
